PHASE 7 — TELEMETRY ENGINE EXTENSIONS
================================================================================
Goal: Add all new TM values needed by Pages 3–8 without changing existing
      fault physics. New values ride on top of existing tick() logic.

Files changed: index.html  (TelemetryEngine class only)

Tasks:

  7.1  Power extensions  (add to nom{}, val{})
         PWR_CELL_V:   array of 6 values, each ~4.0 V ± 0.02 V noise
                       formula: 4.0 + (SOC-80)*0.002 + gauss(0.005) per cell
                       (cells sag together; fault adds extra spread)
         PWR_ECLIPSE:  0 or 1; eclipse when trueAnomaly in [200°,270°]
                       (ISS-like beta angle; ~30% of orbit in shadow)
         PWR_GEN_EFF:  solar array efficiency 0–100 %
                       = 95 - (THERM_PANEL_A + 40) * 0.12 + gauss(0.3)
                       (efficiency drops as panel heats up on daylit side)
         PWR_CHARGE_A: battery charge current (A)
                       = (solar_total - PWR_BUS_LOAD) / PWR_BATT_V
                       (negative when discharging in eclipse)

  7.2  Propulsion extensions
         PROP_FLOW_RATE:  (ml/s)  = _pressLoss-rate * 0.34 + gauss(0.001)
                          (derived from tank pressure drop rate estimate)
         PROP_DV_USED:    (m/s)   = _massLoss / 24.82 * 235
                          (Tsiolkovsky: Isp 235 s, nominal delta-V budget)
         PROP_DV_REMAIN:  (m/s)   = 87.3 - PROP_DV_USED  (budget = 87.3 m/s)
         PROP_THRUSTER_STATUS: array[8] of {id, name, temp, firings, ok}
           Thrusters: A1-A4 (roll/pitch, 1N each), B1-B2 (yaw, 4N each),
                      ME (main engine 22N, rarely fired)
           Temps nominal: 18–24 °C; fault raises temp of A1+A3 by 4 °C

  7.3  Communications extensions
         COMMS_RSSI:     (dBm)  = -87 + sin(met*0.04)*2.1 + gauss(0.3)
                         Goes to -999 when not in contact with any station
         COMMS_BER:      (log10 scale) = -8.2 + gauss(0.2) nominal
                         Rises to -5.8 during link margin degradation
         COMMS_DOPPLER:  (kHz)  = 6.2 * sin(trueAnomaly * DEG2RAD) * cos(INC_R)
                         (peak ±6.2 kHz for 7.66 km/s LEO approaching DSN)
         COMMS_FRAMES_DN: counter, increments 512 per tick when in contact
         COMMS_FRAMES_UP: counter, increments 64 per tick when in contact

  7.4  Data / SSR extensions
         DATA_SSR_FILL:   (%) starts at 34.2, +0.018%/tick (science data accrual)
                          -0.9%/tick while actively downlinking (contact window)
                          clamps 0–100
         DATA_RATE_SCI:   (kbps) = 1024 + gauss(20)   (science instrument rate)
         DATA_RATE_HK:    (kbps) = 64                  (housekeeping, fixed)
         DATA_PRODUCTS:   array[5] of {name, size_mb, status, priority}
           Examples: 'OBS_2026_001', 'CAL_FRAME_44', etc.
           Status cycles: QUEUED → DOWNLINKING → COMPLETE

  7.5  Thruster event log  (this.thrusterEvents = [])
         Push event: { met, thruster, dur_ms, dv_mm_s, cause }
         Fired whenever |roll|>0.5° triggers ADCS correction:
           ADCS fires thruster pair A1+A3 for ~60ms to correct roll
         Fault makes corrections more frequent (every ~8 s vs. ~90 s nominal)
         Keep last 20 events

  7.6  Wiring
         All new vals computed inside tick() after existing fault logic
         All new vals added to nom{} with realistic defaults
         gauss() reused for all noise

Verification:
  - console.log(tm.val) shows all new keys after 1 tick
  - PWR_ECLIPSE toggles between 0 and 1 as trueAnomaly sweeps
  - PROP_DV_REMAIN counts down from ~87.3 m/s
  - COMMS_RSSI is -87 dBm at nominal, spikes during pass windows
  - DATA_SSR_FILL slowly increases at rest, drops during contact window
